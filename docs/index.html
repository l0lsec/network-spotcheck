Network Spotcheck

CLI threat intel for every connection. Bash + Python.

Inventories every active network connection on your machine and checks each remote IP against real threat intelligence. No IP is assumed safe just because it belongs to a known cloud provider.

View on GitHub

Why

Malicious infrastructure routinely lives on AWS, Azure, GCP, and behind Cloudflare. Knowing the provider tells you nothing about whether the traffic is legitimate. This tool queries every IP against actual reputation databases so you get a real answer.

What it does

  • Captures all established connections via lsof — process, PID, remote IP, port.
  • Flags non-standard ports (outside 443, 80, 53, etc.).
  • Resolves reverse DNS for every remote IP.
  • Allowlist — skips API calls for expected process+domain pairs to save quota.
  • Queries VirusTotal and AbuseIPDB for each remaining IP.
  • Flags IPs with non-zero malicious/abuse scores.
  • Sessions — resume interrupted scans; spread large runs across days.

Install

git clone https://github.com/l0lsec/network-spotcheck.git
cd network-spotcheck
chmod +x network_spotcheck.sh   # or: pip install -r requirements.txt && chmod +x network_spotcheck.py

Optional: set VIRUSTOTAL_API_KEY and ABUSEIPDB_API_KEY for live lookups. Without keys, the tool still collects connections, reverse DNS, and port analysis.

Usage

Bash or Python — same options:

./network_spotcheck.sh [OPTIONS]
./network_spotcheck.py [OPTIONS]

Modes

FlagDescription
--mode freeDefault. VT free-tier: 4 req/min, 500/day, 15.5K/month.
--mode premiumNo rate limit (VT premium).
--mode passiveNo API calls — collect + reverse DNS only.

Session & resume

./network_spotcheck.sh --session my_audit
# ... interrupt or quota runs out ...
./network_spotcheck.sh --resume
./network_spotcheck.sh --sessions   # list all

Other options

--show-allowlist · --no-allowlist · --allowlist FILE · --vt-only · --abuse-only · --quota · --reset-quota
